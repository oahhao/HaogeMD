# Markdown 格式规范

## 标题结构

- 标题层级必须连续，严禁跳级使用标题
- `#` 后应先使用 `##`，不得直接跳到 `###`
- `##` 后应先使用 `###`，不得直接跳到 `####`
- 同一层级标题应表达并列关系，不得混用不同抽象层级

## 列表格式

- 无序列表统一使用 `-`
- 嵌套列表缩进 2 个空格
- 列表项表达应保持并列结构
- 列表项不宜过长，过长内容应改为自然段

## 代码块

- 代码块必须指定语言
- Shell 命令使用 `bash`、`powershell` 或对应终端语言
- 配置文件按实际格式标注，例如 `json`、`toml`、`yaml`
- 无法确定语言时使用 `text`
- 代码块前后必须保留空行

## 链接与路径

- 文档内部链接优先使用相对路径
- 外部链接必须使用完整 URL
- 文件路径、命令、变量名、配置项必须使用行内代码格式
- 不得使用失效、猜测或未确认的链接

## 内容维护

- 修改文档时必须同步更新过期示例、路径和命令
- 示例代码必须能表达最小可运行意图
- 不得保留与正文冲突的旧说明
- 无法确认准确性的文档内容必须标注「待验证」
