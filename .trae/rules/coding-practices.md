# 通用编码规范

## 基本原则

- 正确性优先于可读性，可读性优先于简洁，简洁优先于速度
- 优先沿用项目已有结构、命名和技术选型
- 不为局部方便引入新的架构模式、第三方依赖或全局配置
- 不主动重构与当前任务无关的代码

## 控制流

- 避免多层嵌套，优先使用提前返回（Early Return）
- 分支条件应表达业务意图，避免复杂布尔表达式堆叠
- 错误路径优先处理，主流程保持清晰
- 不使用吞异常、空 catch、try/pass 或无意义默认值掩盖问题

## 函数与模块

- 函数只承担一个清晰职责
- 复杂逻辑应拆分为有命名含义的小函数
- 公共逻辑必须先确认已有实现，避免重复造轮子
- 抽象必须服务于当前真实重复，不为假想未来提前设计

## 数据与对象处理

- 避免不必要的对象复制、深拷贝或大数据克隆
- 在安全前提下优先使用引用、切片、迭代器或语言原生高效结构
- 跨边界传输的数据必须有明确类型定义
- 不使用 `any`、弱类型兜底或模糊结构绕过类型约束

## 并发与异步

- 使用语言或框架提供的并发控制机制
- 注意线程安全、资源竞争、锁粒度和取消机制
- 耗时任务不得阻塞 UI 线程或主事件循环
- 异步任务必须有错误处理与状态反馈
